## 2026-05-19 - Native buttons in custom UI fields need manual ARIA attributes\n**Learning:** Highly custom UI fields (e.g., `ResponsiveSliderField`) often use native `<button>` elements with inline SVGs instead of the standard `IconButton` component. These must be manually verified for `aria-label`, `aria-pressed`, and `type="button"`.\n**Action:** When working on custom field components, double-check that raw native buttons have full accessibility context provided, as they don't inherit the automatic visually-hidden label rendering of standard components.
